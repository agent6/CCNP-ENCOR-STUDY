Ch 9 — Advanced OSPF All cheat sheets

ENCOR v1.2 relevance: The second half of 3.2.b — configure simple OSPFv2 environments, including multiple normal areas, summarization, and filtering. Know the LSA types, the ABR type-3 rules (the discontiguous-network trap), intra-vs-inter path preference, and area range.

Areas & ABRs

  • An area = a logical grouping of interfaces (area set per interface; one area per interface). All routers in an area share an identical LSDB; an ABR keeps one LSDB per attached area and computes a separate SPF tree for each.
  • Single-area pain: every link flap = full SPF for everyone, LSDB bloat, zero summarization. Multi-area: topology is hidden outside the area — outside routers do only a partial SPF when metrics change or a prefix disappears. Size areas for the weakest router in them.
  • Area 0 = backbone. All inter-area routes must transit it. An ABR by definition has an interface in Area 0 — a router sitting between two non-backbone areas does NOT pass routes between them.
  • Area ID is 32-bit; decimal (256) and dotted (0.0.1.0) formats are interchangeable and can even differ between neighbors.

Route codes: O intra-area · O IA inter-area (via ABR) · external types exist via redistribution (E1/E2 — light coverage here).

LSA types for IPv4 (six total)

Type Name Who/What Scope
1 Router Every router; lists its OSPF links/networks + metrics Stays inside its area
2 Network The DR of a multi-access segment; lists attached routers Stays inside its area; exists only where a DR was elected; DR change ⇒ new type 2 ⇒ area-wide SPF
3 Summary ABR-created version of another area's networks Regenerated by each ABR it crosses
4 ASBR summary ABR tells other areas how to reach an ASBR —
5 AS external Redistributed routes Domain-wide (except stubby areas)
7 NSSA external Redistribution inside an NSSA Converted to type 5 at the ABR

LSA housekeeping: 32-bit sequence number = version control (higher wins, lower discarded). Age ticks up every second — originator refreshes at 1800 s (re-floods with age 0); anything reaching 3600 s is purged.

Type 3 LSA mechanics (the testable details)

  • ABR receives a type 1 → creates a type 3 with metric = total path cost to the originating router; advertises it into its other areas.
  • ABR receives a type 3 from Area 0 → regenerates a NEW type 3 for the non-backbone area: metric = its cost to the previous ABR + the LSA's metric, and lists itself as advertising router. Receivers see only "the prefix is N away via the ABR" — no topology detail (distance-vector-like behavior between areas).
  • One type 3 per prefix per ABR — best metric only.

The three ABR rules (cause of discontiguous-network failures):

  1. Type 1s from an attached area → type 3s into the backbone AND other attached areas.
  2. Type 3s received from Area 0 → regenerated into non-backbone areas.
  3. Type 3s received from a non-backbone area → used in the source area's LSDB only, never regenerated into other areas (including a split Area 0).

So inter-area traffic cannot transit a non-backbone area: two Area 0 islands glued by Area 23 will exchange nothing past the first ABR. Fix: keep Area 0 contiguous (virtual links/GRE are workarounds; redundancy prevents backbone partition).

Path selection

Priority order — before metric is even compared:

  1. Intra-area (O)
  2. Inter-area (O IA)
  3. External

⚠ An intra-area path with metric 111 beats an inter-area path with metric 21. "OSPF always uses the lowest total metric" = false; lowest metric only breaks ties within the same route type. ECMP installs ties (default 4 paths; maximum-paths to change).

Summarization

  • Happens only at ABRs (LSDBs must stay identical inside an area). Areas themselves are a form of summarization — many type 1/2 LSAs collapse into single type 3s.
  • Config (on the ABR, referencing the SOURCE area of the type 1s): area <area-id> range <network> <mask> [advertise | not-advertise] [cost <metric>]
  • Component (more-specific) type 3s are suppressed; the summary is advertised while ≥1 component exists.
  • Default summary metric = lowest component metric (recomputed when components change — set a static cost to spare the CPU and hide flap churn).
  • The ABR installs a discard route to Null0 for the summary (AD 110 internal / 254 external) — loop prevention for holes in the range.
  • Payoffs: smaller LSDBs, faster SPF, and link flaps inside the summarized range become invisible to other areas (no SPF/partial SPF outside).

Route filtering

Method Command Effect
Summarize-and-hide area X range ... not-advertise No type 3 generated for the range — components visible only in their home area
Area filter-list area X filter-list prefix NAME {in \| out} (on ABR) Prefix-list filtering of type 3 LSAs entering (in) or leaving (out) area X — granular, directional
Local RIB filtering distribute-list ... in Blocks RIB installation on that router only; LSDB/flooding untouched

Prefix-list pattern: deny the target route, then permit 0.0.0.0/0 le 32 to allow the rest.

Exam-day nuggets

  • A router spanning Area 1 + Area 2 (no Area 0) does NOT inject routes between them (DIKTA 1: false). Routers do NOT hold LSDBs for every area — only their own (DIKTA 2: false).
  • Six LSA types route IPv4 (DIKTA 3). LSAs purge at age 3600 s (DIKTA 4). The router (type 1) LSA exists in every area (DIKTA 5).
  • ABRs do NOT forward type 2 (network) LSAs to other areas (DIKTA 6: false). Type 3 from a non-backbone area → installed for the source area only (DIKTA 7).
  • Intra-area beats inter-area regardless of metric (DIKTA 8: false on "always shortest"). Areas = summarization (DIKTA 9: true). Internal summarization = area X range under the process, not summary-address (DIKTA 10).
  • Type 3 metric grows ABR by ABR: cost-to-ABR + LSA metric (worked example: 1 + 65 = 66, then +1 = 67 at the next area).
