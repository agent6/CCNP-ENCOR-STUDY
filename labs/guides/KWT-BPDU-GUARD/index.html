Overview

This lab demonstrates how BPDU Guard protects PortFast edge ports. SW4 has an edge port toward SW5. SW5 starts with its connected interface administratively shut down, so you can first observe normal PortFast behavior, then bring SW5 online to generate a BPDU and force SW4's edge port into err-disabled state.

All commands in this guide use the CML Ethernet interface names shown in the topology.

Topology

SW1 is the root bridge. SW4 connects to the root bridge on Ethernet0/0 and uses Ethernet0/1 as a PortFast edge port toward SW5. SW5 Ethernet0/0 is shut down in the starting configuration.

Task 1: Verify the Baseline

Confirm that SW4 sees Ethernet0/1 as an edge port and that SW5 is not currently sending BPDUs into SW4.

! SW4
show spanning-tree vlan 1
show spanning-tree interface Ethernet0/1 detail
show interfaces Ethernet0/1 status
show spanning-tree summary

Expected result: Ethernet0/1 should appear as a forwarding designated port with edge or PortFast behavior.

Task 2: Observe What Happens When the Edge Neighbor Sends BPDUs

Bring SW5 online and verify that SW4 removes the edge indication from Ethernet0/1 after receiving BPDUs from another switch.

! SW5
conf t
interface Ethernet0/0
 no shutdown
end

! SW4
show spanning-tree vlan 1
show spanning-tree interface Ethernet0/1 detail

Expected result: SW4 Ethernet0/1 remains forwarding, but it should no longer look like a clean host-facing edge port after it receives BPDUs.

Task 3: Restore the PortFast Edge Condition

Shut SW5 back down, then bounce SW4 Ethernet0/1 so the port returns to its PortFast edge behavior.

! SW5
conf t
interface Ethernet0/0
 shutdown
end

! SW4
conf t
interface Ethernet0/1
 shutdown
 no shutdown
end
show spanning-tree vlan 1
show spanning-tree interface Ethernet0/1 detail

Task 4: Compare Interface-Level and Global BPDU Guard

First enable and remove BPDU Guard directly on the interface. Then enable global BPDU Guard so it applies to PortFast edge ports.

! SW4
conf t
interface Ethernet0/1
 spanning-tree bpduguard enable
 no spanning-tree bpduguard enable
exit
spanning-tree portfast edge bpduguard default
end

show spanning-tree summary
show running-config | include bpduguard

Expected result: the summary should show PortFast Edge BPDU Guard Default enabled.

Task 5: Trigger BPDU Guard

Bring SW5 Ethernet0/0 up. SW5 will send BPDUs toward SW4, and SW4 should err-disable Ethernet0/1 because it is a PortFast edge port protected by BPDU Guard.

! SW5
conf t
interface Ethernet0/0
 no shutdown
end

! SW4
show logging | include BPDU|ERR|SPANTREE
show interfaces Ethernet0/1
show interfaces status err-disabled
show spanning-tree vlan 1

Expected result: SW4 Ethernet0/1 should be down/down with an err-disabled reason related to BPDU Guard.

Task 6: Recover the Port

Remove the condition that caused BPDU Guard to fire, then manually recover the port.

! SW5
conf t
interface Ethernet0/0
 shutdown
end

! SW4
conf t
interface Ethernet0/1
 shutdown
 no shutdown
end

show interfaces Ethernet0/1
show spanning-tree interface Ethernet0/1 detail

Optional deeper check: configure and test timed recovery with errdisable recovery cause bpduguard and errdisable recovery interval, then remove those commands when finished.
