Overview

This lab targets ENCOR 3.2.b by configuring simple OSPFv2 and OSPFv3 environments and interpreting neighbor state, router IDs, passive interfaces, and route installation. You will use both network-statement and interface-level styles so you can recognize either in the field.

Topology

Task 1: Baseline Underlay and Router IDs

Confirm all connected interfaces are up and assign stable router IDs. OSPFv3 also needs a router ID even when routing IPv6.

show ip interface brief
show ipv6 interface brief
show ip route connected
show ipv6 route connected
show cdp neighbors detail

Task 2: Configure OSPFv2 Area 0

Use classic OSPFv2 network statements on the backbone, then compare with interface-level ip ospf syntax on one link.

router ospf 10
 router-id 1.1.1.1
 passive-interface default
 no passive-interface ethernet0/0
 network 10.0.12.0 0.0.0.255 area 0
!
interface ethernet0/1
 ip ospf 10 area 0
 ip ospf network point-to-point

Know both configuration styles. Network statements select interfaces by address; interface-level OSPF makes the area membership explicit on the interface.

Task 3: Configure OSPFv3

Enable IPv6 unicast routing and configure OSPFv3 for IPv6. Depending on IOS XE train, you may see classic interface OSPFv3 or address-family OSPFv3.

ipv6 unicast-routing
ipv6 router ospf 10
 router-id 1.1.1.1
 passive-interface default
 no passive-interface ethernet0/0
!
interface ethernet0/0
 ipv6 ospf 10 area 0

Alternative style:

router ospfv3 10
 router-id 1.1.1.1
 address-family ipv6 unicast
  passive-interface default
  no passive-interface ethernet0/0
 exit-address-family
!
interface ethernet0/0
 ospfv3 10 ipv6 area 0

Task 4: Passive Interfaces

Advertise user LANs without sending Hellos toward endpoints. Use passive-interface default and explicitly enable transit links.

show ip ospf interface brief
show ipv6 ospf interface brief
show ip protocols
show ipv6 protocols

A passive OSPF interface still advertises its connected prefix. It simply does not form neighbor adjacencies on that interface.

Task 5: Verification and Defect Repair

  1. Fix a neighbor stuck in INIT because Hellos are not reciprocated.
  2. Fix duplicate router IDs that prevent stable adjacency.
  3. Fix a LAN that is missing from OSPF because it was never selected by the process.
show ip ospf neighbor detail
show ipv6 ospf neighbor detail
show ip route ospf
show ipv6 route ospf
show ip ospf database router
debug ip ospf adj
