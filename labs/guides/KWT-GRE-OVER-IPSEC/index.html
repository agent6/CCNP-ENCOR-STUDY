Overview

This lab builds GRE over IPsec using classic IKEv1, a transform set, a crypto ACL that matches GRE, and a crypto map on the WAN-facing interfaces. The CML topology starts with R1 through R4 underlay links, loopbacks, and OSPF already configured. GRE and IPsec are intentionally absent.

All commands in this guide use the CML interface names shown in the topology.

Topology

R1 and R4 build Tunnel1 across an OSPF-routed underlay through R2 and R3. IPsec protects GRE packets between the public-facing addresses 192.0.2.1 and 198.51.100.2.

Task 1: Verify the Underlay

Start by confirming the underlay is ready before adding GRE or crypto.

! R1
show ip interface brief
show ip ospf neighbor
show ip route ospf
ping 198.51.100.2 source 192.0.2.1

! R4
show ip interface brief
show ip ospf neighbor
show ip route ospf
ping 192.0.2.1 source 198.51.100.2

Task 2: Build the GRE Tunnel

Create Tunnel1 from R1 to R4. The tunnel endpoints are the underlay-facing Ethernet0/0 addresses on R1 and R4.

! R1
conf t
interface Tunnel1
 ip address 192.168.0.1 255.255.255.252
 tunnel source Ethernet0/0
 tunnel destination 198.51.100.2
 no shutdown
end

! R4
conf t
interface Tunnel1
 ip address 192.168.0.2 255.255.255.252
 tunnel source Ethernet0/0
 tunnel destination 192.0.2.1
 no shutdown
end

Verify the tunnel and OSPF adjacency over the tunnel:

show ip interface brief
show interface Tunnel1
show ip ospf neighbor
ping 192.168.0.2 source 192.168.0.1

Task 3: Configure IKEv1 and IPsec on R1

Configure ISAKMP phase 1, the pre-shared key, the transform set, the GRE crypto ACL, and the crypto map on R1.

! R1
conf t
crypto isakmp policy 10
 encryption aes
 authentication pre-share
 group 2
exit
crypto isakmp key kevinskey address 0.0.0.0 0.0.0.0
!
crypto ipsec transform-set KWTRAIN esp-aes esp-sha-hmac
 mode transport
exit
!
ip access-list extended GRE-IN-IPSEC
 permit gre any any
exit
!
crypto map VPN 10 ipsec-isakmp
 match address GRE-IN-IPSEC
 set transform-set KWTRAIN
 set peer 198.51.100.2
exit
!
interface Ethernet0/0
 crypto map VPN
end

Task 4: Configure the Mirrored Policy on R4

Configure the matching IPsec policy on R4 and attach the crypto map to Ethernet0/0.

! R4
conf t
crypto isakmp policy 10
 encryption aes
 authentication pre-share
 group 2
exit
crypto isakmp key kevinskey address 0.0.0.0 0.0.0.0
!
crypto ipsec transform-set KWTRAIN esp-aes esp-sha-hmac
 mode transport
exit
!
ip access-list extended GRE-IN-IPSEC
 permit gre any any
exit
!
crypto map VPN 10 ipsec-isakmp
 match address GRE-IN-IPSEC
 set transform-set KWTRAIN
 set peer 192.0.2.1
exit
!
interface Ethernet0/0
 crypto map VPN
end

Task 5: Verify GRE and IPsec

Generate traffic over the GRE tunnel, then prove that GRE is the interesting traffic being encrypted by IPsec.

! R4
ping 1.1.1.1
show crypto isakmp sa
show crypto ipsec sa
show access-lists GRE-IN-IPSEC
show crypto map

! R1
show crypto isakmp sa
show crypto ipsec sa
show interface Tunnel1
show ip ospf neighbor

Expected result: ISAKMP reaches QM_IDLE, the IPsec SAs are active, and encapsulation/decapsulation counters increment when you send traffic through Tunnel1.
